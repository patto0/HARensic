# 2E — URL Attribution Engine

## Overview

Section **2E** is a new forensic evidence layer added to the HAR Parser For Conversational AI.
It extracts, classifies, and attributes **every URL** involved in an LLM session to one of three
forensic categories using HAR evidence alone — no external lookups required.

---

## Attribution Categories

| Category | Label | Definition |
|---|---|---|
| **HUMAN** | `[H]` | URLs initiated by direct user browser navigation or clicks |
| **AI** | `[A]` | URLs accessed/generated by the model via tool use (search, browsing) |
| **PLATFORM** | `[P]` | URLs belonging to infrastructure: CDN, telemetry, APIs, auth |

---

## Forensic Heuristics

### HUMAN URL Detection
URLs are attributed as HUMAN when:
- HAR `_initiator.type` = `"navigation"` (direct browser navigation)
- HAR `_resourceType` = `"document"` (top-level page load)
- Referer header points to an external non-platform origin
- Request occurs before any `search_start` stream marker
- Domain is not a known platform or third-party content domain

**Examples:** ChatGPT page load, user-opened Google page, manually typed URL

### AI URL Detection ← *Core forensic value*
URLs are attributed as AI when found inside SSE stream metadata:

| SSE Key | Description | Confidence |
|---|---|---|
| `search_result_groups` | Raw search index results returned to AI | 90% |
| `content_references → grouped_webpages` | Pages AI read (with full snippets) | 95% |
| `content_references → sources_footnote` | Final citations shown in UI | 97% |
| `content_references` patch arrays | URL arrays in citation patch deltas | 85-88% |
| `tether_browsing_display` | Legacy Claude/ChatGPT browsing results | 95% |
| `code` content_type blocks | URLs in AI tool invocation code | 92% |

**Key forensic fact:** These URLs **never appear** as direct browser requests in the HAR.
They exist only within streamed assistant metadata, proving server-side AI retrieval.

Confidence is boosted (+3%) when the URL appears after the `search_start` stream marker,
confirming it was generated during AI tool execution.

### PLATFORM URL Detection
URLs are attributed as PLATFORM when:
- Domain exactly matches known platform domains (`chatgpt.com`, `openai.com`, etc.)
- Domain ends with platform infrastructure suffixes (`.oaistatic.com`, `.datadoghq.com`)
- Path matches platform patterns (`/sentinel/`, `/ces/v1/`, `/backend-api/`, etc.)
- URL is a favicon CDN request (reveals AI-cited sites as forensic side-channel)

**Favicon requests** are a special forensic case: attributed PLATFORM (browser UI fetches them)
but contain critical evidence of AI-cited domains embedded in their query parameters.

---

## URL Retrieval Chain

The engine reconstructs the complete AI retrieval pipeline:

```
User Query
    ↓
search_model_queries  (AI's refined search query)
    ↓
search_result_groups  (raw search index: 12 results including irrelevant ones)
    ↓
content_references grouped_webpages  (pages AI actually read + full snippets)
    ↓
sources_footnote  (final citations visible to user)
    ↓
favicon_cdn fetches  (browser UI proof: which sites were cited)
```

This chain is exported as `url_retrieval_chain` in the 2E section.

## Confidence Scoring

| Score | Meaning |
|---|---|
| 97–100% | Definitive — platform domain or sources_footnote citation |
| 90–96% | High — grouped_webpages content or tether_browsing |
| 85–89% | Good — citation patch array or supporting source |
| 70–84% | Moderate — inferred from initiator type |
| 60–69% | Low — default conservative attribution |

## JSON Path Reference

```
SSE stream → data lines (data: {...})
  └── v (dict or list)
       ├── message.content.content_type = "code"           → AI tool call URL
       ├── message.metadata.search_model_queries           → search query
       ├── message.metadata.search_result_groups           → raw results [AI]
       └── message.metadata.content_references             → citations [AI]
            ├── type="sources_footnote" → sources[].url   → final citations [AI]
            └── type="grouped_webpages" → items[].url     → retrieved content [AI]

Network requests → log.entries[].request.url
  ├── _initiator.type = "navigation"                       → HUMAN
  ├── _resourceType = "document"                           → HUMAN
  ├── domain in _PLATFORM_DOMAINS_EXACT                    → PLATFORM (100%)
  └── path matches _PLATFORM_PATH_PATTERNS                 → PLATFORM (97%)

Favicon requests → gstatic.com/faviconV2?url=<cited_domain>
  └── Attribution: PLATFORM
  └── Forensic value: reveals AI-cited domain as side-channel evidence
```

## Output Formats

2E is included in all export formats:

| Format | Notes |
|---|---|
| CLI | Grouped by attribution, role icons, confidence bars |
| JSON | Full artifact rows with all URL metadata fields |
| CSV | `websearch_chatgpt_2E_urls_*.csv` — all extended fields included |
| Markdown | Table in `## URL Attribution & Forensic Evidence` section |
| TXT | Full evidence text per URL |

### Extended CSV/JSON fields for 2E rows

| Field | Description |
|---|---|
| `url_raw` | Original URL before normalization |
| `url_domain` | Extracted domain |
| `url_confidence` | 0–100 confidence score |
| `url_trigger` | What triggered this URL (e.g. `search_start`, `citation_render`) |
| `url_role` | Functional role (e.g. `search_result`, `cited_source`, `cdn_asset`) |
| `url_snippet` | Article snippet if AI read the page |
| `url_pub_date` | Publication date of article (AI URLs) |
| `url_timeline` | ISO timestamp or stream event reference |
| `url_source_type` | `network_request` / `sse_stream` / `derived` |
